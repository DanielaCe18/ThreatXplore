VULNERABILITY 1
WHOIS
The Whois provides detailed information about a registered domain, including the domain's ownership, registration, and expiration dates, contact information for the registrant, administrative, and technical contacts, and the domain's current registrar.
VULNERABILITY 2
Crawler
Browses the internet to discover and download web pages, following links to find new URLs.
VULNERABILITY 3
General information scan
Finds the ip adress of the domain, the http/s transferts, the outgoing links, the cookies, the headers, the outgoing links and the certificate information.
VULNERABILITY 4
SQL injection
SQL Injection is a web security vulnerability that allows an attacker to interfere with the queries that an application makes to its database. This can result in unauthorized access to sensitive data, modification of database content, or even administrative operations on the database.
VULNERABILITY 5
XSS
Cross-Site Scripting (XSS) is a security vulnerability that allows attackers to inject malicious scripts into web pages viewed by other users, potentially stealing sensitive information, hijacking user sessions, or defacing websites.
VULNERABILITY 6
XML EXTERNAL Entity (XXE)
XML External Entity (XXE) is a security vulnerability that occurs when an XML input containing a reference to an external entity is processed by a weakly configured XML parser, leading to potential data exposure, denial of service, or server-side request forgery.
VULNERABILITY 7
OS Command Injection
OS Command Injection is a security vulnerability that occurs when an application allows untrusted input to be executed as part of a command or query, enabling attackers to execute arbitrary commands on the host operating system.
VULNERABILITY 8
CORS Vulnerability
A CORS (Cross-Origin Resource Sharing) vulnerability arises when a web applicationâ€™s CORS policy is misconfigured, allowing unauthorized domains to access restricted resources, which can lead to data theft or unauthorized actions.
VULNERABILITY 9
Server-Side Template Injection (SSTI)
Server-Side Template Injection (SSTI) is a security vulnerability that occurs when user input is unsafely incorporated into a server-side template, enabling attackers to execute arbitrary code on the server.
VULNERABILITY 10
Email and Credit Card Disclosure
Email and Credit Card Disclosure is a security vulnerability where sensitive information, such as email addresses and credit card details, is exposed to unauthorized individuals, leading to privacy breaches and potential financial fraud.
VULNERABILITY 11
Server-Side Request Forgery (SSRF)
Server-Side Request Forgery (SSRF) is a security vulnerability that occurs when an attacker is able to induce a server-side application to make unauthorized requests to arbitrary domains, potentially leading to data exposure, internal network scanning, or interaction with internal services.
VULNERABILITY 12
Cross-Site Request Forgery (CSRF)
Cross-Site Request Forgery (CSRF) is a security vulnerability that forces an authenticated user to perform unwanted actions on a web application in which they are currently authenticated, potentially leading to unauthorized state changes or data manipulation.
VULNERABILITY 13
Uncommon HTTP Methods
Uncommon HTTP Methods vulnerability occurs when a web server or application improperly handles or does not restrict uncommon HTTP methods like PUT, DELETE, TRACE, or OPTIONS, potentially leading to unauthorized actions or information disclosure.
VULNERABILITY 14
HTTP Redirection Vulnerability
HTTP Redirection Vulnerability occurs when an application improperly handles redirection requests, allowing attackers to redirect users to malicious sites, leading to phishing attacks or further exploitation.
VULNERABILITY 15
HTTP Security Headers Missing
The lack of HTTP security headers is a vulnerability where a web application fails to implement headers that enhance security, such as Content-Security-Policy, X-Content-Type-Options, X-Frame-Options, and Strict-Transport-Security, making the application more susceptible to attacks like XSS, clickjacking, and MIME type sniffing.
VULNERABILITY 16
Robots.txt Availability Vulnerability
Robots.txt Availability Vulnerability occurs when sensitive or critical URLs are inadvertently exposed through the robots.txt file, which is intended to guide web crawlers but can be accessed by attackers to discover hidden parts of the application.
VULNERABILITY 17
Local File Inclusion (LFI)
Local File Inclusion (LFI) is a security vulnerability that occurs when an application allows users to include files from the local server through manipulated input, potentially leading to arbitrary file access, sensitive data exposure, or remote code execution.
VULNERABILITY 18
Unrestricted File Upload
Unrestricted File Upload is a security vulnerability that occurs when an application does not properly validate or restrict the types of files that users can upload, potentially allowing attackers to upload malicious files that can be executed on the server.
VULNERABILITY 19
Path Traversal
Path Traversal is a security vulnerability that occurs when an application improperly validates user input, allowing attackers to access files and directories outside the intended directory, potentially leading to sensitive data exposure or execution of arbitrary code.
VULNERABILITY 20
Open Ports Vulnerability
Open Ports Vulnerability occurs when a system has unnecessary or poorly secured network ports open, which can be exploited by attackers to gain unauthorized access or perform malicious activities.
VULNERABILITY 21
WebSocket Manipulation Vulnerability
WebSocket Manipulation Vulnerability occurs when an application improperly handles WebSocket communications, allowing attackers to intercept, modify, or inject messages, leading to unauthorized actions, data theft, or further exploitation.
VULNERABILITY 22
Certificate Issues
Certificate validation is not properly configured, allowing attackers to use invalid, expired, or untrusted certificates to spoof legitimate services.
VULNERABILITY 23
SSL/TLS vulnerabilities
SSL/TLS security issues include supported protocols, weak ciphers, Perfect Forward Secrecy, and various known vulnerabilities like Heartbleed, POODLE, BEAST, CRIME/BREACH, along with checks for HSTS implementation and TLS Fallback SCSV support.
VULNERABILITY 24
Common Password Vulnerability
Weak or predictable passwords that can be easily guessed or cracked.
VULNERABILITY 25
Brute Force Vulnerability
Attackers attempt to gain access by systematically trying all possible combinations of passwords.
VULNERABILITY 26
Account Lockout Vulnerability
Users can be locked out of their accounts after a number of failed login attempts, which can be exploited to cause denial of service.
